CRITICAL: Full-Resolution Image Publicly Accessible via Google Cloud Storage

The imgix signed URLs only protect the CDN layer. The original source images in Google Cloud Storage are publicly readable by anyone who knows (or guesses) the path.

Proof

# Preview served by the page (14 KB, 400x225, q=40):
https://cmgcreate-2.imgix.net/social/7106fc31-3457-4923-af95-680dd9a2720e/title.jpg
  ?auto=format&fit=crop&h=225&q=40&w=400&s=SIGNED

# FULL RESOLUTION directly from GCS (5.8 MB, 2000x2500, original quality):
https://storage.googleapis.com/cmgcreate/social/7106fc31-3457-4923-af95-680dd9a2720e/title.jpg

# File size comparison:
#   Preview: 14,265 bytes (14 KB)
#   Original: 5,959,885 bytes (5.8 MB) — 418x larger!
#   Original dimensions: 2000 x 2500 pixels

How an attacker finds this

  1. View page source → find imgix URL containing cmgcreate-2.imgix.net
  2. Note the bucket name is cmgcreate (also exposed in window.CONSTANTS.googleStorageBucket)
  3. Extract the path: social/7106fc31-.../title.jpg
  4. Construct GCS URL: https://storage.googleapis.com/cmgcreate/[path]
  5. Download the full 2000x2500 original — no authentication needed

Fix Required (Server-Side)

# 1. Make GCS bucket private (remove allUsers/allAuthenticatedUsers read access):
gsutil iam ch -d allUsers:objectViewer gs://cmgcreate
gsutil iam ch -d allAuthenticatedUsers:objectViewer gs://cmgcreate

# 2. Configure imgix to use a service account for GCS access
#    (imgix will still work, but direct GCS URLs will return 403)

# 3. Remove googleStorageBucket from window.CONSTANTS
#    (currently exposed in constants.js: "googleStorageBucket":"cmgcreate")

All Ways People Can Steal Images

  • Google Cloud Storage direct access — CRITICAL: full-res originals are public (see above)
  • Right-click > Save Image As — most common method
  • Drag image to desktop — works on most browsers
  • View Page Source — find <img src> or og:image meta tag
  • DevTools > Network tab — filter by images, copy URL
  • DevTools > Elements tab — inspect, find src attribute
  • Derive GCS URL from imgix path — bucket name exposed in constants.js
  • Screenshot / Screen capture — OS-level, impossible to fully prevent
  • cURL/wget — download via command line if URL is known
  • Browser extensions — "Download All Images" type addons
  • Disable JavaScript — bypasses all JS-based protections
  • Google/Bing Image Cache — search engines cache og:image

Protection Techniques (All Methods)

CSS 1. Disable Right-Click Context Menu

// Disable right-click on the entire page or specific elements
document.addEventListener('contextmenu', function(e) {
    e.preventDefault();
    return false;
});

CSS 2. Transparent Overlay

Place an invisible div over the image. Right-click hits the overlay, not the image.

.protected-image-wrapper {
    position: relative;
}
.protected-image-wrapper::after {
    content: '';
    position: absolute;
    top: 0; left: 0; right: 0; bottom: 0;
    z-index: 10;
    background: transparent;
}

CSS 3. Use CSS background-image Instead of <img>

No <img> tag = no "Save Image As" in context menu.

.preview-image {
    width: 400px;
    height: 225px;
    background-image: url('your-image.jpg');
    background-size: cover;
    background-position: center;
}

CSS 4. Disable Drag & Select

img {
    -webkit-user-drag: none;
    user-drag: none;
    -webkit-user-select: none;
    user-select: none;
    pointer-events: none;
}

JS 5. Render Image on Canvas

Draw image onto a <canvas>. The original URL is never exposed in the DOM.

const canvas = document.getElementById('protectedCanvas');
const ctx = canvas.getContext('2d');
const img = new Image();
img.crossOrigin = 'anonymous';
img.onload = () => ctx.drawImage(img, 0, 0, canvas.width, canvas.height);
img.src = 'your-image-url.jpg';

JS 6. Disable Keyboard Shortcuts

document.addEventListener('keydown', function(e) {
    // Block Ctrl+S (Save), Ctrl+U (View Source), Ctrl+Shift+I (DevTools)
    if ((e.ctrlKey && (e.key === 's' || e.key === 'u')) ||
        (e.ctrlKey && e.shiftKey && e.key === 'I') ||
        e.key === 'F12' ||
        (e.ctrlKey && e.shiftKey && e.key === 'J')) {
        e.preventDefault();
        return false;
    }
});

HTML 7. Remove og:image Meta Tag for Previews

The og:image meta tag exposes the direct image URL. Use a low-res/watermarked version instead.

<!-- BAD: Exposes full image URL -->
<meta property="og:image" content="https://cdn.example.com/full-image.jpg">

<!-- GOOD: Use a watermarked/low-res version -->
<meta property="og:image" content="https://cdn.example.com/preview-watermarked.jpg">

SERVER 8. Signed/Expiring URLs (Already in use!)

Your imgix URLs already use signed parameters (&s=...). Unsigned URLs return 403. This is great!

# Your current URL (signed, time-limited):
https://cmgcreate-2.imgix.net/social/.../title.jpg?auto=format&fit=crop&h=225&q=40&w=400&s=SIGNATURE

# Without signature → 403 Forbidden (already working!)

SERVER 9. Serve Low-Quality Previews Only

You're already serving q=40 (low quality) at 400x225. Anyone who steals this gets a tiny, blurry image. Consider going lower.

# Current: q=40, 400x225 (decent protection)
# Better:  q=20, 300x169 (even more degraded)
# Add visible watermark via imgix:
?auto=format&fit=crop&h=225&q=30&w=400&mark=watermark.png&mark-w=200&mark-align=center,middle

SERVER 10. imgix Watermarking

Use imgix's built-in watermark parameter to burn a watermark into the served image:

# Add watermark overlay via imgix URL params:
?mark=https://yoursite.com/watermark.png&mark-w=300&mark-align=center,middle&mark-alpha=30

SERVER 11. Hotlink Protection (Referer Check)

Configure your CDN/server to only serve images when the Referer header matches your domain.

# Nginx example:
location ~* \.(jpg|jpeg|png|gif|webp)$ {
    valid_referers none blocked ~\.churchmotiongraphics\.com;
    if ($invalid_referer) {
        return 403;
    }
}

// Hide images when printing
@media print {
    .protected-image-wrapper { display: none !important; }
    body::before {
        content: 'Image printing is disabled.';
        display: block;

Recommended Protection Stack for Your Site

No single technique is bulletproof. Use these together for maximum protection:

  1. Keep signed imgix URLs (already done) — prevents URL manipulation
  2. Keep low quality previews (q=40, 400px) — stolen image is useless
  3. Add imgix watermark — burns "PREVIEW" into the image server-side
  4. Use CSS background-image or Canvas — removes "Save Image As" option
  5. Add transparent overlay + disable right-click — stops casual theft
  6. Disable drag — prevents drag-to-desktop
  7. Use watermarked image for og:image — protects social sharing preview
  8. Add hotlink protection — prevents embedding on other sites

Reality check: Determined users can always screenshot. The goal is to make theft inconvenient enough that people buy instead. Low-quality + watermark + no easy download = most people will just purchase.
